Adding components
Updated on 15 Sept 2021
In ADCM a component means a part of a service, that should be deployed on one or several cluster hosts.
For example, HDFS service can consist of such components, as: DataNode, NameNode, Client, and others.
IMPORTANT
Initially, there are no components on any of cluster hosts — the distribution of components between hosts should be performed manually.
The steps of adding components to hosts are listed below:
  1. From the tab CLUSTERS switch to cluster configuring. To do this, click at the row with the added cluster or click at the icon in the column Config. Both methods lead to the opening of the cluster menu.
  2. Open the tab Hosts - Components in the cluster menu.
    NOTE
    The orange exclamation mark near the menu item Hosts - Components is displayed, when components distribution is needed.
  3. In the opened page distribute all necessary components between cluster hosts. To distribute components, use one of the ways:
    1. Select a component in the Components column (by clicking at it) and then define all hosts for it in the Hosts column (by alternate clicking at each host).
    2. Select a host in the Hosts column (by clicking at it) and define all components for it in the Components column (by alternate clicking at each component).

Mandatory and optional components

Each service can have mandatory and optional components:
  • If the component is optional, it has a simple quantitative counter, placed to the right of its name — without the symbol "/". This counter displays the number of hosts, assigned to the current component.
  • If the component is mandatory, its quantitative counter contains the symbol "/". The first digit in the counter (displayed before "/") shows the current number of assigned hosts, and the second digit (displayed after "/") — the minimal required number of hosts.
CAUTION
Without assigning necessary count of hosts to mandatory components, the service distribution map cannot be saved.
