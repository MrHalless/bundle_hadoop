ADCM installation

Updated on 15 Sept 2021
ADCM requires a separate host. Before installing ADCM, we recommend to read its documentation.
The installation steps are listed below.

Step 1. Checking prerequisites

Software and hardware requirements

ADCM is a software, distributed in the form of a Docker Image. To install it, you need:
  • A maintained version of CentOS 7/RHEL 7 operation system. Archived versions are not supported or tested.
  • A user account with sudo privileges.
  • Access to the official repository of CentOS Extras and CentOS Base. This repository is enabled by default, but if you have disabled it, you need to re-enable it.
  • Software package installer YUM.
    • CPU: 2-4 CPU Cores
    • RAM: 8-16 GB
    • HDD: 8-50 GB

Docker installation

CAUTION
Do not install Docker from the official Docker site. Instead of it, use official repositories of your Linux distribution. Repositories, provided by operation system developers, tend to be more stable and secure.
The typical installation sequence via the package manager YUM includes the following steps:
  1. Using root privileges, update all packages, currently installed in your system: 
    $ sudo yum update

SeLinux disabling

SELinux should be disabled for ADCM. To do it permanently, set SELINUX=disabled in the file /etc/selinux/config and restart your system.
You can edit the file, using vi or vim command: 
$ vi /etc/selinux/config

Copied
The content of the changed file should look like this: 

            # This file controls the state of SELinux on the system.
            # SELINUX= can take one of these three values:
            #       enforcing - SELinux security policy is enforced.
            #       permissive - SELinux prints warnings instead of enforcing.
            #       disabled - SELinux is fully disabled.
            SELINUX=disabled
            # SELINUXTYPE= type of policy in use. Possible values are:
            #       targeted - Only targeted network daemons are protected.
            #       strict - Full SELinux protection.
            SELINUXTYPE=targeted

Step 2. Installing ADCM

IMPORTANT
Before installing ADCM make sure, that:
  • You have installed some Docker or compatible container engine for running ADCM.
  • You have created your own SELinux policies or switched it off. Notice, that current software version is not compatible with SELinux.
To install ADCM you should run the following commands:
  1. Pull a Docker Image from Arenadata Docker Registry. 
    $ docker pull hub.arenadata.io/adcm/adcm:latest
    Copy
    Copied
  2. Create a container, based on that image. The command below makes a container on the port 8000 with ADCM. All its data will be stored on the host machine in the directory /opt/adcm/. 
    $ docker create --name adcm -p 8000:8000 -v /opt/adcm:/adcm/data arenadata/adcm:latest
    Copy
    Copied
NOTE
ADCM has some mandatory persistent information, which is stored in /adcm/data directory inside of a container. That means, that you have to map a volume to that directory and provide for that volume backups.

Step 3. Starting ADCM

ADCM is just a container, operated by Docker. So you are able to use simple Docker commands to start and stop ADCM:
  1. To start ADCM use the following command: 
    $ sudo docker start adcm

Checking ADCM web-interface

To log in to ADCM, you should use the user with administrator rights, which is created by default:
  • User name: admin
  • Password: admin
Later you can change the password, if necessary. To do this:
  1. Click the list item Settings in the account menu.
  2. Select the tab Users.
  3. Change the password and confirm it in the according fields.

Checking ADCM URL

If ADCM container ends up on the network using NAT, it makes impossible the automatic determining the address of ADCM container, which is used by clusters components for sending information about their state to ADCM.
In this and other cases of complicated network structures, you may need to specify ADCM URL manually. To do this:
  1. Click the list item Settings in the account menu.
  2. Select the tab Settings.
  3. Check the correctness of the automatically determined URL in the field ADCM’s URL and replace it with a valid one, if necessary.
  4. Click Save.
